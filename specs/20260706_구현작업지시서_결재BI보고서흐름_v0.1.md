## 1. 기능 요약 (무엇을 만드나)

**요구**: ① 홈에서 업무의 결재 프로세스 확인 → ② 문제점을 BI로 → ③ 보고서까지 단일 흐름. ④ BI는 구체적·"있어보이는" 항목(정합성 불요). ⑤ **AI 1–2줄 요약 상시 노출**. ⑥ 리치한 UI.

```
[홈 CM-03] 업무 노드 클릭 ─ 결재상태 칩(회색/노랑/초록/빨강)·Drawer 결재 미니 요약(AI 1줄+미니 스텝)
   ▼ [결재 상세 보기]                                  (INT-CM03-13~15)
[WF-02 워크플로우 탭] 결재선 타임라인(기안→검토→승인)·병목/지연 빨강·AI 요약 배너
   ▼ [BI로 원인 분석]                                  (INT-WF02-11~14)
[RB-01 BI] B1~B9 위젯(리드타임·병목 히트맵·반려율·SLA·추세·원인·부하·예측)+카드별 ✦ AI 한 줄
   ▼ [보고서 작성]                                     (INT-RB01-03~05)
[RB-03 보고서] 결재 이력+BI 문제 반영 초안 자동생성(문제=빨강) → 편집 → 발송(승인 HITL)
                                                      (INT-RB03-04~05)
```

각 단계는 다음 단계 CTA로 자연 연결(역방향 링크 포함). **모든 관련 뷰 상단에 `✦` AI 요약 1–2줄 상시**(mock 문장·§4.3).

## 4. 데이터·mock 명세 (정식 정의 — 임의 필드 금지)

### 4.5 시드 보장 조건 (시연 케이스)

- **지연 업무 ≥1**: dueAt 경과+elapsedDays>평균 (예: '예산요구서' 검토 5일) → WF-02 빨강·B3 1위·홈 배지 노랑/빨강.
- **반려 업무 ≥1**: REJECTED+rejectReason (예: MTG 연계 업무 1건) → 홈 칩 빨강·미니 스텝 risk·AI 요약 반려 문구.
- **riskItems 연계**: 결재 지연 업무를 riskItems(type=병목/일정)와 `refId`로 연결. notifications `category=결재` 활용.
- 나머지 업무는 DRAFTING/APPROVING/APPROVED 분포(자연스럽게).

## 5. 색상·토큰 규칙 (DEF-05 — 위반 시 FAIL)

- `design/tokens.css`의 `--color-*`만 사용. **hex 하드코딩 금지.**
- **빨강(risk)=문제 전용·희소**(반려·지연·기준미달·위험만). 순수 진행률·정보성엔 금지.
- 솔리드 배지=빨강·초록·회색만, **노랑(warn)은 틴트형**.
- 결재상태·단계·위험등급·게이지 적용은 §4.2 표와 specs 각 INT 블록 기준.

## 6. 검증 워크플로 (REF-13/15)

1. Phase 시작 전 **플랜 모드**로 계획·영향 파일·검증 전략 승인.
2. **1 INT = 1 구현 = 1 증거**: 스크린샷/테스트/콘솔 — "됐다" 주장 금지. Playwright(MCP)로 트리거→결과 시연·캡처 권장.
3. INT 완료 시 spec의 **수용기준(GWT) 대조** 후 커밋(`feat: INT-…`).
4. Phase 종료 시 `/verify-interaction-spec <SCR>` → 전 INT 행 DONE/MISSING/WRONG 체크.
5. **spec-validator 서브에이전트**(코드 수정 금지·문제만 보고)로 독립 검증 → PASS 후 다음 Phase.
6. 명세에 없는 값·동작은 구현하지 말고 `[CONFIRM: 질문]`으로 보고.
