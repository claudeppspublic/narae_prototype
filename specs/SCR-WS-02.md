# SCR-WS-02 — 조직도 · 조직도뷰 (델타 스펙)

> **델타 스펙** — 화면 전체 명세가 아닌, v0.9 to-be 추가분만 수록한다.
> 근거: `docs/20260703_화면상세정의서_v0.9.xlsx` §4 개선요구(to-be) — "조직 구성원에 대한 노드까지 출력 / 임시값이라도 넣어서 정보를 더 표시할 수 있도록 노드 디자인 변경" (우선순위 0, 미반영→본 스펙으로 반영)
> 구현 파일: `src/pages/org/OrgChart.jsx` · `src/components/canvas/CanvasNodes.jsx`

## 1. 범위

조직도뷰 캔버스에서 말단 조직(과) 전개 시 **구성원(worker) 노드**를 출력하고, 노드에 입출력정의서 USER 스키마 기반 정보를 확장 표시한다.

## 2. 데이터 항목 (입출력데이터정의서 v0.9 · USER 스키마)

| 필드 | 타입 | 표시 |
|------|------|------|
| empNo | string | 사번 |
| empNm | string | 성명 (아바타 이니셜 포함) |
| position | string | 직위 |
| employmentStatus | enum | 재직/휴직/퇴직 뱃지 (`EMPLOYMENT_STATUS`) |
| jobTags | array | 사무분장 태그 (최대 3개 + 초과분 `+N`) |
| (파생) taskCount | number | 담당 업무 건수 — `tasksByAssignee(empNo).length` |

## 4. 버튼·인터랙션

| INT-ID | 요소 | 트리거 | 동작 | 결과/피드백 | 전제조건 | 예외/비활성 |
|--------|------|--------|------|-------------|----------|-------------|
| INT-WS02-01 | 말단 조직(과) 노드 | 클릭 | 하위 조직이 없으면 `usersOf(orgUnitId)` 구성원을 worker 노드로 전개 | 구성원 노드가 해당 조직에 엣지로 연결되어 표시 | 말단 조직(childrenOf=0) | 소속 인원 0명 → 전개 없음 |
| INT-WS02-02 | 구성원(worker) 노드 | 렌더 | USER 스키마 정보 확장 표시 | 성명·직위 + 사번(empNo) + 재직상태 뱃지 + 업무 N건 + 사무분장 태그(≤3, 초과 `+N`) | INT-WS02-01로 전개됨 | 태그 없음 → 태그 영역 미표시 |

## 5. 수용기준 (Given-When-Then)

- **INT-WS02-01**: Given 조직도뷰에서 하위 조직이 없는 과 노드가 보일 때 When 해당 노드를 클릭하면 Then 소속 구성원 노드들이 과 노드 우측에 전개된다.
- **INT-WS02-02**: Given 구성원 노드가 전개되었을 때 When 노드를 보면 Then 성명·직위·사번·재직상태·업무 건수가 표시되고, 사무분장 태그가 있으면 최대 3개까지(초과분은 `+N`) 표시된다.

## 6. 스타일

- 색·간격은 `var(--color-*)` / `var(--krds-*)` / `var(--narae-*)` 토큰만 사용.
- 재직상태 뱃지: 재직=ok 계열, 휴직=warn 계열, 퇴직=neutral 계열 토큰 + 텍스트 라벨(색 단독 구분 금지).
