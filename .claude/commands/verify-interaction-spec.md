# /verify-interaction-spec

인자로 받은 `<SCR-ID>`의 `specs/SCR-<SCR-ID>.md`에서 인터랙션 표(§4 버튼·인터랙션)의 모든 행을 추출하고,
해당 화면의 구현 코드(`src/pages/` 내 대응 파일)와 대조해 각 행을 아래 상태로 분류한다:

| 상태 | 의미 |
|------|------|
| ✅ DONE | 트리거·동작·결과가 코드에 정확히 구현됨 |
| ❌ MISSING | 해당 인터랙션이 코드에 없음 |
| ⚠️ WRONG | 구현은 있으나 스펙과 불일치 (트리거/결과/전제조건 등) |

## 점검 항목
각 INT 행에 대해:
1. **트리거** — 해당 이벤트 핸들러가 코드에 존재하는가?
2. **동작** — 트리거 시 스펙에 명시된 동작이 실행되는가?
3. **결과/피드백** — 상태 변경, UI 갱신, 네비게이션 등이 스펙대로인가?
4. **전제조건** — 역할 제한, 데이터 존재 여부 등 전제가 코드에 반영되었는가?
5. **예외** — 에러/비활성/빈상태 등 예외 처리가 구현되었는가?

## 추가 점검
- 상태 5종 (초기/로딩/데이터/빈/에러) 구현 여부
- mock 키가 스펙의 데이터 항목과 일치하는지
- `[CONFIRM]` 미해소 항목 잔존 여부

## 출력 형식
```
## SCR-XX-XX 검증 결과

| # | INT-ID | 트리거 | 상태 | 비고 |
|---|--------|--------|------|------|
| 1 | INT-XX-01 | 클릭 | ✅ DONE | |
| 2 | INT-XX-02 | 입력 | ❌ MISSING | onChange 핸들러 없음 |

상태 5종: ✅/❌
mock 키 일치: ✅/❌
[CONFIRM] 잔존: 없음/있음(N건)
```
